## Why

現在のタイムラインでは、各メッセージの右上 `...` 操作ボタンを表示するためだけに専用ヘッダー行が確保され、本文の前に常に空行が入っている。メッセージ一覧の情報密度と視認性を下げているため、通常表示時に不要な余白を生まないレイアウトへ見直す必要がある。

## What Changes

- メッセージ項目のレイアウト要件を更新し、`...` 操作導線のためだけの専用1行を確保しない表示に変更する。
- メッセージ本文はカード上端から自然に読み始められる配置とし、各メッセージ上部の不要な空行を解消する。
- 操作メニュー（編集・削除）の導線と挙動は維持しつつ、モバイル/PC の両表示幅で視認性を損なわないことを要件化する。

## Capabilities

### New Capabilities

- なし

### Modified Capabilities

- `message-timeline-view`: メッセージ操作メニュー導線の配置要件を更新し、通常表示時にメッセージ本文の前へ不要な空行を発生させないことを追加する。

## Impact

- **フロントエンド**:
  - `frontend/src/components/MessageList.tsx` のメッセージ項目マークアップを調整
  - `frontend/src/App.css` のメッセージカード/メニュー周辺スタイルを調整
  - `frontend/src/components/MessageList.test.tsx` と `frontend/src/App.test.tsx` の表示回帰テストを更新
- **バックエンド / API**:
  - 変更なし（既存のメッセージ API 契約は維持）
