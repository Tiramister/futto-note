## Why

ログイン機能まで実装した段階では、保存済みノートを参照する手段がなく、アプリの主要価値である「記録した内容を後から確認する」体験が成立していない。Step 3 として一覧取得・表示を追加し、ノート閲覧の基本フローを完成させる必要がある。

## What Changes

- バックエンドに認証必須の `GET /api/messages` を追加し、ログインユーザーのメッセージを `created_at` 昇順で返却する
- フロントエンドのメイン画面を、メッセージ一覧領域と下部入力エリアを持つタイムラインレイアウトに更新する
- メッセージ表示に本文と投稿時刻を含め、本文中の URL をクリック可能なリンクとして表示する
- 初回表示時の最下部スクロールと、スマホ/PC 両方で崩れないレスポンシブ表示を追加する

## Capabilities

### New Capabilities

- `message-list-api`: 認証済みユーザー向けにメッセージ一覧を取得する API の要件（取得対象、並び順、レスポンス形式）を定義する
- `message-timeline-view`: メッセージ一覧の UI 表示要件（タイムライン構造、URL 自動リンク化、初期スクロール、レスポンシブ対応）を定義する

### Modified Capabilities

- なし

## Impact

- **バックエンド**:
  - `backend/main.go` に `GET /api/messages` ハンドラーとルーティングを追加
  - `messages` テーブルの読み取りクエリを追加
- **フロントエンド**:
  - `frontend/src/App.tsx` のメイン画面をタイムライン表示へ更新
  - メッセージ取得処理と URL リンク化表示ロジックを追加
  - レスポンシブ用スタイルを `frontend/src/App.css` へ追加
- **API**:
  - 新規エンドポイント `GET /api/messages` を公開
